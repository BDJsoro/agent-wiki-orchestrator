# Antigravity Agent Wiki Orchestrator 

이 패키지는 옵시디언(Obsidian) 환경에서 동작하는 독립적인 **AI 에이전트 오케스트레이터(Antigravity LLM Wiki Orchestrator)** 템플릿입니다. 개인 기록이나 특정 스크립트를 제외한 순수 에이전트 시스템만이 포함되어 있어 누구나 쉽게 설치하고 자신의 환경에 적용할 수 있습니다.

## 📦 구성 요소

- `AGENTS.md`: 최상위 오케스트레이터 지휘소. 에이전트 시스템의 전역 규칙(Global Constraints)과 YAML 메타데이터 스키마가 정의되어 있습니다.
- `env.md`: 에이전트가 분류에 참고할 범주(Category) 등 전역 환경 변수를 설정합니다.
- `skills/`: `librarian`, `linter`, `challenger`, `connector`, `emerger`, `architect`, `synthesizer` 등 각 에이전트 위원회의 개별 행동 지침서가 모여있습니다.
- `workflows/`: 지식 처리, RAG 루프 등 에이전트가 실행할 작업별 표준 템플릿과 프로세스.

## 🚀 설치 및 적용 방법

1. 여러분이 사용하고 계신 **옵시디언 Vault(작업 폴더 최상위) 경로**를 엽니다.
2. 이 패키지 안에 있는 `.agents` 폴더를 복사하여, 여러분의 Vault 최상위에 그대로 붙여넣습니다.
3. **(중요)** AI에게 **"`.agents/AGENTS.md`의 3번째 줄에 있는 예전 작업 경로를 내 현재 작업 공간(Workspace) 경로로 수정해 줘"** 라고 요청하거나 직접 수정하세요.
4. `env.md` 파일을 열고, 그 안에 선언된 `ALLOWED_CATEGORIES` 등을 여러분의 프로젝트 목적에 맞게 수정합니다.
5. 이제 AI가 `.agents/AGENTS.md` 파일을 기반으로 전체 지휘 체계를 인식하게 됩니다.

## ⚙️ 작동 원리 (Inbox Zero)

* 수집된 원본 문서를 `inbox/` 폴더에 넣습니다.
* 에이전트가 `AGENTS.md`의 규칙에 따라 원본 문서의 데이터를 분석하고 YAML 메타데이터를 주입합니다.
* 처리된 원본 문서는 `library/sources/`로 이관되며, 에이전트 위원회의 토론 결과가 반영된 새로운 지식 페이지가 `library/wiki/`에 자동으로 생성됩니다.

---

## ⚡ 빠른 시작 (Quick Start)

1. `inbox/` 폴더에 처리하고 싶은 문서(`.md`, `.txt` 등)를 넣습니다.
2. 이 폴더를 작업 공간으로 열고 있는 AI(예: Antigravity, Cursor AI, Claude 등)에게 다음과 같이 입력합니다:
   > `$doit`
3. AI가 Pre-flight 체크리스트를 자동 수행하고, 7명의 위원회 토론을 거쳐 `library/wiki/`에 지식 노드를 자동으로 생성합니다.

| 명령어 | 설명 |
|---|---|
| `$doit` | inbox 폴더가 빌 때까지 모든 파일을 순차 처리 |
| `$gp` | 처리 후 지식 그래프 시각화 출력 요청 |

## 🎯 카테고리 커스터마이징 (중요!)

`.agents/env.md`의 `ALLOWED_CATEGORIES` 목록은 **현재 HRD/리더십/마케팅 분야에 특화된 예시 값**이 들어 있습니다.  
**자신의 분야에 맞는 카테고리로 교체해야** 에이전트가 올바르게 지식을 분류합니다.

예시:
- 개발자라면: `[알고리즘, 아키텍처, 디버깅, API설계, 코드리뷰, ...]`
- 투자자라면: `[가치투자, 리스크관리, 포트폴리오, 시장분석, ...]`
- 교육자라면: `[수업설계, 평가, 동기부여, 피드백, 교육과정, ...]`
